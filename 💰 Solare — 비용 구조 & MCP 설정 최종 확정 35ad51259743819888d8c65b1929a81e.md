# 💰 Solare — 비용 구조 & MCP 설정 최종 확정

## 핵심 요약

```
데이터: 각 사용자 본인 Drive + 본인 Firebase
개발자 서버: MCP 서버(Railway) 하나뿐
개발자 월 비용: ₩0 (MCP Keep-Alive 조절 시)
```

---

## 1. 데이터 개인화 구조 (방법 A 확정)

```
사용자 A
  └─ A의 Google Drive (모든 데이터 저장)
  └─ A의 Firebase   (협업 시 본인이 직접 설정)

사용자 B
  └─ B의 Google Drive
  └─ B의 Firebase

개발자(당신)
  └─ Firebase 없음
  └─ 사용자 서버 없음
  └─ 사용자 데이터 접근 불가
```

10만 명이 사용해도 당신에게 Firebase 비용 없음.

---

## 2. Firebase 동시 100명 제한 — 정확한 의미

```
❌ 오해: 전체 사용자 100명 제한
✅ 정확: 하나의 Firebase 프로젝트에
         동시 연결된 소켓 100개 초과 시

방법 A에서는:
  각 사용자가 본인 Firebase를 씀
  → A의 Firebase: A + A가 초대한 팀원만
  → B의 Firebase: B + B가 초대한 팀원만
  → 한 팀이 동시에 100명 협업할 일 없음
  → 사실상 제한 없음
```

## 6. MCP 서버 활성 시간 — 최종 확정 (C안)

```
09:00 ~ 13:00  (4시간)
13:00 ~ 17:00  (4시간)
20:00 ~ 02:00  (6시간, 자정 넘김)

하루 총 14시간
월 사용량: 14 × 31 = 434시간
Railway 무료 한도: 500시간
여유분: 66시간 ✅
비용: ₩0
```

### 02:00 이후 작업하면 어떻게 되나

```
01:59 작업 중
        ↓
02:00 됨
        ↓
Keep-Alive ping 중단
        ↓
15분간 요청 없음 감지
        ↓
02:15경 슬립 진입

→ 02:00 땡하자마자 끊기지 않음
→ 진행 중인 작업 정상 완료
→ 02:15 이후 새 요청 시 30초 Cold Start
```

## 7. 구현 코드

### MCP 서버 인증 + 시간 체크

```jsx
const ACTIVE_PERIODS = [
  { start: 9,  end: 13 },   // 09:00 ~ 13:00
  { start: 13, end: 17 },   // 13:00 ~ 17:00
  { start: 20, end: 26 },   // 20:00 ~ 02:00 (26 = 다음날 02:00)
];

app.use((req, res, next) => {
  // 인증 체크
  const key = req.headers['x-mcp-key'];
  if (key !== process.env.MCP_SECRET_KEY) {
    return res.status(401).json({ message: '인증 실패' });
  }

  // 시간 체크 (KST 기준)
  const now = new Date();
  const kst = (now.getUTCHours() + 9) % 24;
  const kstDecimal = kst + now.getUTCMinutes() / 60;
  const adjusted = kstDecimal < 3 ? kstDecimal + 24 : kstDecimal;

  const active = ACTIVE_PERIODS.some(p =>
    adjusted >= p.start && adjusted < p.end
  );

  if (!active) {
    return res.status(503).json({
      message: 'MCP 서버 비활성 시간입니다.',
      activePeriods: '09~13, 13~17, 20~02 KST'
    });
  }
  next();
});
```

### GitHub Actions Keep-Alive (KST → UTC 변환)

```yaml
# .github/workflows/mcp-keep-alive.yml
name: MCP Keep-Alive

on:
  schedule:
    # KST 09:00~17:00 = UTC 00:00~08:00
    - cron: '*/10 0,1,2,3,4,5,6,7 * * *'
    # KST 20:00~02:00 = UTC 11:00~17:00
    - cron: '*/10 11,12,13,14,15,16 * * *'

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping MCP Server
        run: |
          curl -s \
            -H "x-mcp-key: ${{ secrets.MCP_SECRET_KEY }}" \
            https://forgewrite-mcp.railway.app/health
```

비활성 시간에는 ping 없음 → Railway 자연스럽게 슬립 진입.
