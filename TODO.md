# Solare 확인 투두리스트

> 빌드 → 로그인 → 각 기능 순서대로 진행
> 완료 시 `[ ]` → `[x]` 로 변경

---

## 빌드

- [ ] `flutter build web --release --base-href /Solare/` 오류 없음
- [ ] `flutter build windows --release --dart-define=DESKTOP_CLIENT_SECRET=...` 오류 없음

---

## 로그인

- [ ] 웹 — Google 로그인 팝업 뜨고 로그인 완료 → `/projects`로 이동
- [ ] 웹 — 새로고침해도 로그인 유지 (자동 재인증)
- [ ] exe — 로그인 클릭 → 시스템 브라우저 열림 → 앱 자동 인증
- [ ] exe — 앱 재시작해도 로그인 유지

---

## 프로젝트

- [ ] 프로젝트 생성 → 목록에 표시
- [ ] 앱 재시작 → 프로젝트 목록 그대로 (Drive에서 복원)
- [ ] 프로젝트 클릭 → 기능 카드 그리드 (Notes/Branch Tree/캐릭터/위키/에피소드/타임라인)

---

## 노트

- [ ] 새 페이지 만들기 → 목록에 추가
- [ ] 내용 입력 → 3초 후 자동 저장
- [ ] 하위 페이지 만들기 → 트리에서 중첩 표시
- [ ] 앱 재시작 → 노트 내용 유지

---

## 분기 수형도

- [ ] 빈 캔버스 클릭 → 노드 추가 다이얼로그
- [ ] 타입별 색상/모양 다름 (event=파랑/사각, choice=주황/마름모, condition=보라/평행사변형, result=초록/타원)
- [ ] 노드 드래그 → 위치 이동
- [ ] 노드 테두리 핸들 드래그 → 다른 노드로 화살표 연결
- [ ] 노드 클릭 → 연결된 노트 페이지 열림

---

## 캐릭터

- [ ] 캐릭터 추가 → 목록에 표시
- [ ] 기본 정보 탭 — 이름/MBTI/배경 이야기 저장
- [ ] 습관 추가/삭제, 트라우마 추가/삭제
- [ ] 관계 탭 — 다른 캐릭터 선택 + 관계 타입 + 설명 추가
- [ ] 앱 재시작 → 캐릭터 데이터 유지

---

## 위키

- [ ] 항목 추가 (카테고리 선택) → 카테고리별 그룹으로 표시
- [ ] 내용 입력 → 3초 후 자동 저장
- [ ] `[[다른항목]]` 입력 → 저장 후 링크 아이콘에 숫자 배지
- [ ] 링크 아이콘 클릭 → 링크 목록 팝업
- [ ] 존재하는 항목 링크 클릭 → 해당 항목으로 이동
- [ ] 없는 항목 링크 → 주황색 경고 아이콘

---

## 에피소드

- [ ] 에피소드 추가 (제목 + 회차 번호) → 회차 순 정렬
- [ ] 내용 입력 → 하단 글자 수 실시간 업데이트
- [ ] 3초 후 자동 저장

---

## 무드보드 (exe 전용)

- [ ] 웹에서 접근 시 "PC 앱에서 사용 가능" 안내 표시
- [ ] exe — 텍스트 블록 추가 → 캔버스에 표시
- [ ] exe — 색상 블록 추가
- [ ] exe — 블록 드래그 → 위치 이동
- [ ] exe — 블록 길게 누르기 → 삭제

---

## 타임라인

- [ ] 이벤트 추가 (제목 + 날짜 + 정렬순서) → 순서대로 표시
- [ ] 날짜 자유 형식 ("Day 1", "3년 2월", "2025-03-01" 전부 입력 가능)
- [ ] 이벤트 편집/삭제
- [ ] 앱 재시작 → 타임라인 유지

---

## 협업 (선택, 두 기기 필요)

- [ ] 협업 프로젝트 생성 → 팀원 이메일 초대
- [ ] 상대방 앱에서 초대 알림 팝업
- [ ] 수락 → 동일 프로젝트 접근
- [ ] 한쪽 편집 → 상대방 화면 3초 내 반영

---

## 배포

- [ ] `git push origin main` → GitHub Actions 웹 배포 자동 실행
- [ ] `https://j-mug.github.io/Solare/` 접속 → 앱 정상 표시
- [ ] `git tag v1.0.0 && git push origin v1.0.0` → Windows 빌드 + Releases 업로드
- [ ] zip 다운로드 → 압축 해제 → `solare.exe` 실행 정상
