## 1. 이 플랫폼이 무엇인가

### 세 가지 도구가 하나로

```
📝 노션형 노트 시스템
  페이지 안에 페이지 (무한 중첩)
  마크다운 블록 에디터
  일상 노트 + 창작 설정 정리 모두 가능
          +
🗡️ 창작 도구 (ForgeTales 스타일)
  캐릭터 시트 / 세계관 위키
  무드보드 / 타임라인
  에피소드 에디터 (소설 집필)
          +
🎮 게임 설계 도구
  분기 수형도 (동적 노드 그래프)
  선택 → 결과 조건 연결
```

## 4. MVP 범위 (첫 번째로 만들 것)

Hemisphere 작업에 당장 필요한 두 모듈부터 시작.

### MVP 모듈 1: 노트 시스템

```
페이지
  ├─ 제목
  ├─ 텍스트 블록
  ├─ 제목 블록 (H1/H2/H3)
  ├─ 체크리스트 블록
  ├─ 인용 블록
  ├─ 구분선
  ├─ 코드 블록
  └─ 하위 페이지 블록 ← 페이지 안에 페이지
       └─ 또 다른 페이지 (무한 반복)
```

블록 에디터 `/` 명령어로 블록 타입 선택. 드래그로 순서 변경.

### MVP 모듈 2: 분기 수형도

```
[노드 생성]
  - 이벤트 노드 (사건/장면)
  - 선택지 노드 (플레이어 선택)
  - 조건 노드 (Route 완료 여부 등)
  - 결과 노드 (세계 변화, 아이템 획득)

[노드 연결]
  - 노드 간 화살표로 연결
  - 조건 태그 부착 (예: "Route 2 완료 필수")
  - 노드 클릭 → 연결된 노트 페이지 열기

[뷰 조작]
  - 전체 트리 확대/축소/이동
  - 미니맵
  - 특정 노드 검색
```

Hemisphere의 현재 ASCII 플로우차트를 이 수형도로 대체.

### 사용자 경험

```

        ↓
"Google로 시작하기" 버튼 (딱 한 번만)
        ↓

  ✅ 계정 기본 정보
  ✅ 앱 전용 Drive 폴더 접근 (숨김 폴더)
        ↓
허용
        ↓
이후 모든 실행: 자동 로그인 (버튼 없음)
토큰 만료 시: 백그라운드에서 자동 갱신
```


## 6. 실시간 협업 시스템

### 전체 구조

```
Firebase Realtime Database (중계자 역할)
        ↑↓                    ↑↓
     내 앱               초대받은 사람 앱
        ↓                         ↓
  Google Drive              Google Drive
  (내 원본 저장)            (공유된 폴더)
```

### 초대 프로세스 (자동화)

```
내가 앱에서 이메일 입력 → "초대" 클릭
        ↓
앱이 자동으로:
  ① Drive API: 프로젝트 폴더 공유 권한 부여
  ② Firebase: 멤버 목록에 등록
  ③ 초대 이메일 자동 발송
        ↓
상대방이 이메일 링크 클릭
  → 앱 열림 → Google 로그인
  → 자동으로 프로젝트 접근
  → "이 파일도 공유하시겠습니까?" 없음
        ↓
실시간 협업 시작
```

### 데이터 흐름 (텍스트/노드)

```yaml
내가 타이핑 / 노드 이동
        ↓ 즉시 (수십 ms)
Firebase에 변경사항(delta)만 전송
        ↓                    ↓
Drive에 저장          상대방 화면 즉시 반영
(3초 후 debounce)
        ↓
Firebase delta 삭제 (용량 비움)
```

### 데이터 흐름 (이미지/파일)

```
이미지 추가
        ↓
Drive에 직접 업로드 (수 초~수십 초)
Firebase엔 "파일 ID 바뀜" 한 줄만 전송
        ↓
상대방이 "파일 ID 바뀜" 수신
        ↓
Drive에서 해당 파일 다운로드
로딩 표시 → 완료되면 자동 표시
```

### Firebase 1GB 용량 관리

```
Firebase에 저장하는 것        실제 용량
━━━━━━━━━━━━━━━━━━━━━━        ━━━━━━━━━━━
텍스트 변경 delta만           수십~수백 바이트/건
Drive 저장 후 즉시 삭제       → 누적 데이터 0에 수렴
이미지는 Firebase 경유 안 함  → 1GB 사실상 무한

활성 사용자 10명 동시 작업:
10명 × 100바이트 × 60초 = 60KB/분
Drive 저장 후 삭제 → 거의 안 참
```

### Drive 변경 감지 (Drive Watch API)

```
Drive 파일 변경 발생
        ↓
Google이 앱에 자동 Push 알림
        ↓
앱이 해당 파일만 다시 로드
(주기적으로 직접 확인할 필요 없음)
```

## 7. 동기화 엔진 전체 로직

```dart
class SyncEngine {

  // 앱 시작 시
  Future<void> onAppStart() async {
    // 1. Drive에서 전체 원본 로드
    final remote = await drive.getManifest();
    final local = await db.getManifest();

    if (remote.version > local.version) {
      await pullFromDrive();       // 원격 → 로컬
    } else if (queue.isNotEmpty) {
      await pushToDrive();         // 로컬 → 원격
    }

    // 2. Firebase 실시간 구독 시작
    firebase.subscribe(projectId, onDelta);
  }

  // 텍스트/노드 편집 시
  Future<void> onEdit(Change change) async {
    await db.save(change);         // 즉시 로컬 저장
    await firebase.sendDelta(change); // Firebase로 즉시 전송 (실시간)

    // Debounce: 3초 후 Drive에 저장
    debounce(3000, () async {
      await pushToDrive();
      await firebase.deleteDelta(change.id); // Firebase 정리
    });
  }

  // 이미지 추가 시
  Future<void> onImageAdd(File image) async {
    final fileId = await drive.uploadFile(image); // Drive 직접 업로드
    await firebase.sendDelta({'type': 'image', 'fileId': fileId}); // ID만 전송
  }

  // 상대방 변경사항 수신 시
  void onDelta(Delta delta) {
    if (delta.type == 'image') {
      drive.downloadFile(delta.fileId); // Drive에서 파일 가져오기
      showLoadingIndicator();           // 로딩 표시
    } else {
      db.apply(delta);                  // 즉시 화면 반영
    }
  }

  // 충돌 해결 (타임스탬프 기준)
  Change resolveConflict(Change local, Change remote) {
    return remote.updatedAt > local.updatedAt ? remote : local;
  }
}
```

```yaml
# 인증 & Drive
google_sign_in: ^6.x

# 로컬 DB (오프라인 지원)

```

### 프로젝트별 Firebase 자동 전환

```

  → 팀원 초대 → 모두 홍길동 Firebase 사용
```

### 알림 시스템 구조

초대 알림을 위해 **ForgeWrite 공용 Firebase 하나**가 필요합니다.

이것만 개발자(당신)가 운영하는 유일한 Firebase입니다.

```
ForgeWrite 공용 Firebase (개발자 운영)
  역할: 초대 알림 전달만 담당
  저장 데이터: 초대장 (수락 후 즉시 삭제)

        (초대장은 수백 바이트, 수락 후 삭제)

  역할: 실시간 협업 delta 중계
  저장 데이터: delta (Drive 저장 후 삭제)
```

```
공용 Firebase 데이터 구조
  invites/

        status, created_at
        ↓
  수락/거절 후 즉시 삭제
```

### 보안

Firebase API Key는 클라이언트에 공개되어도 되는 값입니다.

실제 보안은 Firebase Rules가 담당합니다.

```jsx
// 오너 Firebase Rules
// 멤버로 등록된 uid만 접근 가능
{
  "rules": {
    "projects": {
      "$project_id": {
        ".read": "root.child('members')
                      .child($project_id)
                      .hasChild(auth.uid)",
        ".write": "root.child('members')
                       .child($project_id)
                       .hasChild(auth.uid)"
      }
    },
    "members": {
      "$project_id": {
        ".write": "auth.uid === root.child('projects')
                        .child($project_id)
                        .child('owner_uid').val()"
      }
    }
  }
}
```

수락 버튼을 누른 사람만 멤버로 등록됩니다.

초대장을 다른 사람이 가로채도 Rules에서 차단됩니다.

### 프롬프트 5 — 동기화 엔진

```
Drive + Firebase를 조합한 동기화 엔진을 구현해줘.

[동작 방식]
1. 앱 시작 시
   - Drive manifest.json 버전 확인
   - 원격이 최신이면 pull, 로컬 큐 있으면 push
   - Firebase delta 구독 시작

2. 편집 발생 시
   - 로컬 SQLite에 즉시 저장
   - Firebase에 delta 즉시 전송 (실시간)
   - 3초 debounce 후 Drive에 저장
   - Drive 저장 완료 후 Firebase delta 삭제
```

### 데이터 개인화 구조

```
사용자 A
  └─ A의 Google Drive (데이터 저장)
  └─ A의 Firebase (협업 시, 본인이 직접 설정)

사용자 B
  └─ B의 Google Drive (데이터 저장)
  └─ B의 Firebase (협업 시, 본인이 직접 설정)

개발자(당신)
  └─ Firebase 없음
  └─ 서버 없음
  └─ 사용자 데이터 접근 불가
```

**10만 명이 사용해도 당신에게 Firebase 비용 없음.**

```
  각 사용자가 본인 Firebase를 씀

  → 사용자 B의 Firebase: B와 B가 초대한 팀원만 접속
```

### MCP 서버 Keep-Alive 시간 설정

```
  │                                     │
  │  시작: [09:00 ▼]                   │
  │  종료: [24:00 ▼]                   │

```

### MCP 서버 나만 사용 (인증 설정)

```
Railway 환경변수
  MCP_SECRET_KEY = 내가_정한_비밀키

Claude 앱 설정
  x-mcp-key 헤더에 동일한 키 입력

→ 키가 맞으면 연결
→ 틀리거나 없으면 401 차단
→ Railway URL을 알아도 키 없으면 접근 불가
```

**인증 + 시간 체크 통합 로직**

```jsx
// 인증
app.use((req, res, next) => {
  const key = req.headers['x-mcp-key'];
  if (key !== process.env.MCP_SECRET_KEY) {
    return res.status(401).json({ message: '인증 실패' });
  }

  // 시간 체크 (KST 기준)
  const now = new Date();
  const kst = (now.getUTCHours() + 9) % 24;
  const kstDecimal = kst + now.getUTCMinutes() / 60;
  const adjusted = kstDecimal < 3 ? kstDecimal + 24 : kstDecimal;

  const ACTIVE = [
    { start: 9,  end: 13 },
    { start: 13, end: 17 },
    { start: 20, end: 26 },
  ];

  const active = ACTIVE.some(p =>
    adjusted >= p.start && adjusted < p.end
  );

  if (!active) {
    return res.status(503).json({
      message: 'MCP 서버 비활성 시간입니다.',
      activePeriods: '09~13, 13~17, 20~02 KST'
    });
  }
  next();
});
```

```
→ 원본 수정 시 참조한 곳도 자동 업데이트
→ 읽기 전용 (수정하려면 복사 후 편집)
```

**⑩ 공개 포트폴리오 페이지**

```
프로젝트 설정에서 "일부 공개" 선택
  → 공개할 섹션 체크 (위키만, 캐릭터만 등)
  → 고유 링크 생성
     forgewrite.github.io/share/abc123

방문자는 읽기 전용으로 열람
  → 계정 없이도 볼 수 있음
  → 분기 수형도도 인터랙티브하게 탐색 가능

GitHub Pages의 별도 정적 뷰어로 구현
(별도 서버 없이 Drive 데이터를 읽기 전용으로 렌더링)
```

## 16. 템플릿 시스템 상세 설계

### 템플릿이란

프로젝트의 구조(노트 계층, 분기 수형도 형태, 캐릭터 시트 양식 등)는 가져오되, 실제 내용(텍스트, 설정값)은 비워두거나 예시로 채운 재사용 가능한 시작점.

```
예시 템플릿
  📦 TRPG 분기 게임 템플릿
    ├─ 📝 노트
    │    ├─ 세계관 설정 (빈 양식)
    │    ├─ 등장인물 목록 (예시 1명)
    │    └─ 루트 설계 가이드
    ├─ 🎮 분기 수형도
    │    └─ 기본 3분기 구조 (시작→선택A/B/C→결말)
    └─ 🧑 캐릭터 시트
         └─ 기본 시트 양식
```

---

### 공유 방식: 링크 공유 (즉시) + GitHub 저장소 (성장 후)

#### 1단계: 링크 공유 방식 (Phase 4 초기)

```
[템플릿 만드는 사람]

1. 프로젝트 완성
2. "템플릿으로 내보내기" 클릭
3. 내용을 비워두고 구조만 남기기 (옵션)
4. Drive의 공개 공유 폴더에 template.json 저장
5. 공유 링크 복사
   예: https://drive.google.com/file/d/abc123
6. 커뮤니티(Discord, X, 블로그)에 링크 공유

[템플릿 받는 사람]

1. 앱의 "새 프로젝트" → "템플릿에서 시작"
2. "링크로 가져오기" 클릭
3. 링크 붙여넣기
4. 앱이 Drive에서 template.json 다운로드
5. 내 Drive의 새 프로젝트로 복사
6. 바로 편집 시작
```

## 14. Claude MCP 연동 시스템

### 개요

Claude 데스크탑/모바일 앱의 커넥터(MCP) 기능을 통해 ForgeWrite 데이터에 Claude가 직접 접근. 사용자가 Claude 앱에서 자연어로 요청하면 Claude가 Google Drive의 프로젝트 데이터를 읽고 창작 보조, 분기 분석, 노트 정리를 수행.

### 전체 구조

```
Claude 데스크탑 / Claude 모바일 앱
        ↓ MCP 프로토콜
ForgeWrite MCP 서버 (Railway에 배포)
        ↓ Google Drive API
사용자 Drive (appDataFolder)
        ↓ 데이터 반환
Claude가 읽고 분석 → 답변
```

### MCP 서버 호스팅: Railway + Keep-Alive

```
Railway 무료 티어
  월 500시간 실행
  메모리 512MB
  슬립: 15분 미사용 시 진입
  Cold Start: 20~30초

+ GitHub Actions Keep-Alive
  10분마다 자동 핑 전송
  → 슬립 방지 → Cold Start 완전 제거
  → 월 Actions 사용량: 약 1,440분 (무료 2,000분 이내)

최종 비용: ₩0
```

**Keep-Alive 설정**

```yaml
# .github/workflows/keep-alive.yml
name: MCP Keep-Alive

on:
  schedule:
    - cron: '*/10 * * * *'  # 10분마다

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping MCP Server
        run: curl -s https://forgewrite-mcp.railway.app/health
```

```tsx
    { name: 'analyze_branch_consistency', description: '분기 일관성 검토' },

  ]
}));

// 툴 실행
server.setRequestHandler('tools/call', async ({ name, arguments: args }) => {
  const drive = new GoogleDriveClient(args.access_token);

  switch (name) {
    case 'read_branch_tree':
      const tree = await drive.readFile(
        `projects/${args.project_id}/branch_tree.json`
      );
      return { content: [{ type: 'text', text: JSON.stringify(tree) }] };

    case 'analyze_branch_consistency':
      const data = await drive.readFile(
        `projects/${args.project_id}/branch_tree.json`
      );
      const issues = analyzeConsistency(data);
      return { content: [{ type: 'text', text: JSON.stringify(issues) }] };

    // ... 나머지 툴들
  }
});
```
